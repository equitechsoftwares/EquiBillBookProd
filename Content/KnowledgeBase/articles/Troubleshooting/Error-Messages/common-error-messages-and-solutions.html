Common Error Messages and Solutions

This guide provides solutions for common error messages you may encounter while using EquiBillBook. Each error message is explained with step-by-step solutions.

Authentication and Access Errors

"Invalid email or password"

Solution:

  • Verify you're using the correct email address
  • Check if Caps Lock is enabled (passwords are case-sensitive)
  • Use the "Forgot Password" link to reset your password
  • Clear browser cache and cookies
  • Try logging in from a different browser

"Account locked" or "Too many login attempts"

Solution:

  • Wait 15-30 minutes before attempting to log in again
  • Use the "Forgot Password" feature to reset your password
  • Contact support if account remains locked

"Access denied" or "You don't have permission"

Solution:

  • Verify you have the required permissions for this action
  • Contact your administrator to grant necessary permissions
  • Check if your user account is active
  • Review user role and permission settings

Data Entry and Validation Errors

"Required field is missing" or "Field cannot be empty"

Solution:

  • Fill in all required fields marked with asterisk (*)
  • Check for validation error messages near specific fields
  • Review form to ensure all mandatory information is provided

"Invalid data format" or "Invalid format"

Solution:

  • Check the expected format for the field (date, number, email, etc.)
  • Verify date format matches system settings (DD/MM/YYYY or MM/DD/YYYY)
  • Ensure numbers don't contain invalid characters
  • Check email format is correct (user@domain.com)

"Duplicate entry" or "Already exists"

Solution:

  • Check if the record already exists in the system
  • Use a unique identifier (code, number, etc.)
  • Search for existing records before creating new ones
  • Edit existing record instead of creating duplicate

"Invalid date range" or "Start date must be before end date"

Solution:

  • Ensure start date is earlier than end date
  • Check date format is correct
  • Verify dates are within valid range

Transaction Errors

"Insufficient stock" or "Stock not available"

Solution:

  • Check current stock level for the item
  • Reduce quantity to available stock level
  • Create purchase order to increase stock
  • Check if stock tracking is enabled for the item
  • Verify stock at correct location (if multi-location)

"Invoice already paid" or "Payment exceeds amount"

Solution:

  • Check invoice outstanding balance
  • Verify payment amount doesn't exceed invoice amount
  • Review previous payments for the invoice
  • Create partial payment if amount is less than total

"Cannot delete - transaction in use" or "Record in use"

Solution:

  • Check if transaction is linked to other transactions
  • Cancel or delete dependent transactions first
  • Review transaction status (may need to cancel before deleting)
  • Check if transaction is referenced in reports

"Cannot edit - transaction confirmed" or "Transaction locked"

Solution:

  • Cancel the transaction first if editing is required
  • Create a new transaction with corrections
  • Use credit note or adjustment for corrections
  • Check user permissions for editing confirmed transactions

File and Upload Errors

"File too large" or "File size exceeds limit"

Solution:

  • Reduce file size (compress images, reduce document size)
  • Check maximum file size limit in settings
  • Use file compression tools if needed
  • Split large files if possible

"Invalid file type" or "File type not supported"

Solution:

  • Check supported file formats for the upload
  • Convert file to supported format
  • Verify file extension matches file type
  • Common formats: JPG, PNG, PDF, XLSX, CSV

"Upload failed" or "File upload error"

Solution:

  • Check internet connection stability
  • Try uploading again
  • Verify file isn't corrupted
  • Check browser compatibility
  • Clear browser cache and try again

Configuration and Settings Errors

"Settings not saved" or "Failed to save settings"

Solution:

  • Verify all required fields are filled
  • Check for validation error messages
  • Ensure you have permission to modify settings
  • Try saving in different browser
  • Clear browser cache and try again

"Invalid configuration" or "Configuration error"

Solution:

  • Review configuration settings for errors
  • Verify all required configuration is complete
  • Check configuration documentation
  • Reset to default settings if needed

Payment and Gateway Errors

"Payment gateway error" or "Gateway connection failed"

Solution:

  • Verify payment gateway is properly configured
  • Check API credentials are correct
  • Verify payment gateway service is operational
  • Test payment gateway connection
  • Contact payment gateway support if issue persists

"Payment declined" or "Transaction failed"

Solution:

  • Verify card details are correct
  • Check if card has sufficient funds
  • Ensure card is active and not expired
  • Contact bank to verify card status
  • Try different payment method

Email and SMS Errors

"Email sending failed" or "SMTP error"

Solution:

  • Verify SMTP settings are correct
  • Check email credentials (username, password)
  • Test SMTP connection in settings
  • Verify email service is operational
  • Check if email sending limits are exceeded

"SMS sending failed" or "SMS gateway error"

Solution:

  • Verify SMS gateway is configured
  • Check SMS gateway API credentials
  • Verify SMS balance/credits are available
  • Check phone number format is correct
  • Test SMS gateway connection

Report and Export Errors

"Report generation failed" or "Unable to generate report"

Solution:

  • Verify date range is valid
  • Check if data exists for selected criteria
  • Reduce date range if report is too large
  • Apply filters to reduce data volume
  • Try generating report again

"Export failed" or "Unable to export"

Solution:

  • Try different export format (PDF, Excel, CSV)
  • Reduce data range for export
  • Check available disk space
  • Verify browser download settings
  • Try exporting from different browser

Network and Connection Errors

"Connection timeout" or "Request timeout"

Solution:

  • Check internet connection stability
  • Try again after a few moments
  • Reduce data volume in request
  • Check firewall or antivirus settings
  • Try from different network

"Network error" or "Unable to connect"

Solution:

  • Verify internet connection is working
  • Check if EquiBillBook service is operational
  • Try refreshing the page
  • Clear browser cache
  • Disable VPN if using one

General Error Handling Tips

When You See an Error:

  1. Read the Error Message: Error messages often contain helpful information
  2. Note Error Details: Copy or screenshot the complete error message
  3. Check Related Settings: Review settings related to the error
  4. Try Basic Solutions: Refresh page, clear cache, try different browser
  5. Review Recent Changes: Check if recent changes caused the issue
  6. Search Knowledge Base: Search for similar errors in help documentation
  7. Contact Support: If issue persists, contact support with error details

Preventing Common Errors

  • Validate Data: Always verify data before saving
  • Check Permissions: Ensure you have required permissions
  • Review Settings: Regularly review and verify configuration settings
  • Keep Updated: Ensure system and browser are up to date
  • Backup Data: Regularly backup important data
  • Test Changes: Test changes in test environment when possible

Getting Help

If you encounter an error not listed here or if solutions don't work:

  • Document the Error: Note exact error message, when it occurred, and what you were doing
  • Check Logs: Review system logs if accessible
  • Search Knowledge Base: Search for similar issues in help documentation
  • Contact Support: Provide:
    • Complete error message
    • Steps to reproduce the error
    • Screenshots if possible
    • Browser and version
    • Any recent changes made

Most errors can be resolved by following the solutions provided. If an error persists, don't hesitate to contact EquiBillBook support for assistance.
