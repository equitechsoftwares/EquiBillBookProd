Restaurant Configuration

Restaurant Configuration in EquiBillBook Settings allows you to set up core restaurant operations, preferences, and default behaviors. This comprehensive guide covers all configuration options available in Restaurant Settings.

Accessing Restaurant Configuration

To configure restaurant settings:

  1. Navigate to Settings from the main menu
  2. Click on Restaurant Settings
  3. You'll see the "Restaurant Configuration" section
  4. Configure various options as needed

General Restaurant Configuration

Restaurant Name and Details

Set up basic restaurant information:

  • Restaurant Name: Display name for your restaurant
  • Restaurant Description: Brief description of your restaurant
  • Restaurant Type: Type of restaurant (Fine Dining, Casual, Fast Food, etc.)
  • Cuisine Type: Primary cuisine served
  • Contact Information: Restaurant-specific contact details

Default Settings

Configure default behaviors:

  • Default Booking Duration: Standard booking time (e.g., 2 hours)
  • Default Table Capacity: Default number of seats per table
  • Auto-confirm Bookings: Automatically confirm bookings or require approval
  • Default Deposit Percentage: Standard deposit amount as percentage of estimated bill
  • Default Currency: Currency for restaurant transactions

Booking Configuration

Advance Booking Settings

  • Maximum Advance Booking Days: How far in advance customers can book (e.g., 30, 60, 90 days)
  • Minimum Advance Booking Hours: Minimum hours before booking time (e.g., 2 hours)
  • Same-day Booking: Allow or restrict same-day bookings
  • Booking Buffer Time: Time between consecutive bookings on same table

Booking Duration Settings

  • Minimum Booking Duration: Shortest allowed booking time
  • Maximum Booking Duration: Longest allowed booking time
  • Default Booking Duration: Standard booking length
  • Extended Booking: Allow bookings beyond maximum duration with approval

Multiple Table Booking

  • Allow Multiple Tables: Enable customers to book multiple tables in one booking
  • Maximum Tables per Booking: Limit on number of tables per booking
  • Adjacent Table Requirement: Require multiple tables to be adjacent

Deposit Configuration

Deposit Requirements

  • Deposit Required: Enable or disable deposit requirements
  • Deposit Type: Fixed amount or percentage of estimated bill
  • Deposit Amount: Fixed deposit amount (if using fixed type)
  • Deposit Percentage: Percentage of estimated bill (if using percentage type)
  • Minimum Deposit: Minimum deposit amount regardless of percentage
  • Maximum Deposit: Maximum deposit amount cap

Deposit Payment

  • Deposit Payment Methods: Which payment methods accept deposits
  • Online Deposit Payment: Allow online payment for deposits
  • Deposit Account: Account where deposits are recorded
  • Deposit Refund Policy: Automatic or manual refund processing

Notification Configuration

Booking Notifications

  • New Booking Alerts: Notify staff when new bookings are created
  • Booking Confirmation: Send confirmation emails/SMS to customers
  • Booking Reminders: Send reminders before booking time
  • Booking Cancellation Alerts: Notify when bookings are cancelled
  • Notification Recipients: Who receives booking notifications

KOT Notifications

  • New KOT Alerts: Notify kitchen when new orders are placed
  • KOT Status Updates: Notify when order status changes
  • Kitchen Display: Enable real-time kitchen display system

Table Management Configuration

Table Status Settings

  • Auto Status Update: Automatically update table status based on bookings
  • Status Update Rules: Rules for when status changes occur
  • Maintenance Mode: How maintenance mode works for tables
  • Table Capacity Rules: Rules for table capacity and overbooking

Waiter Assignment

  • Auto Waiter Assignment: Automatically assign waiters to tables
  • Assignment Rules: Rules for waiter assignment (round-robin, availability, etc.)
  • Default Waiter: Default waiter for new bookings

KOT Configuration

KOT Settings

  • Auto-print KOT: Automatically print KOT when orders are placed
  • KOT Numbering: KOT numbering sequence and prefixes
  • Kitchen Display: Enable kitchen display system
  • Station Assignment: Automatic or manual kitchen station assignment
  • Item Status Tracking: Enable item-level status tracking in KOT

Public Booking Configuration

Portal Settings

  • Enable Public Booking: Allow customers to book through public portal
  • Booking URL: Custom URL for booking portal
  • Require Registration: Require customers to create accounts
  • Show Available Tables: Display available tables on portal
  • Allow Online Payment: Enable payment processing through portal

QR Code Configuration

QR Code Settings

  • Enable Table QR Codes: Generate QR codes for each table
  • QR Code Actions: What happens when QR codes are scanned
  • QR Code Format: QR code design and format
  • QR Code Size: Size of generated QR codes

Saving Configuration

After configuring all settings:

  1. Review all changes you've made
  2. Click "Save" or "Update Settings"
  3. The system will confirm that settings have been saved
  4. Settings will take effect immediately

Best Practices

  • Configure Before Use: Set up all configurations before creating bookings
  • Test Settings: Test booking and KOT features with sample data
  • Review Regularly: Periodically review and update configurations
  • Balance Convenience: Balance customer convenience with business needs
  • Document Settings: Keep notes of your configuration choices

Related Settings

After configuring Restaurant Configuration, you should also set up:

  • Operating Hours: Set your restaurant's operating schedule
  • Date Overrides: Configure special dates with different hours
  • Public Booking Settings: Fine-tune public booking portal
  • QR Code Settings: Configure QR code functionality

Proper restaurant configuration ensures smooth operations and optimal customer experience!
