Purchase Return Workflow

The purchase return workflow in EquiBillBook guides you through the complete process of returning items to suppliers, from identification of the need to return through final reconciliation. This guide explains the step-by-step workflow and best practices.

Overview of Purchase Return Workflow

The purchase return workflow consists of several stages:

  1. Identification: Identifying items that need to be returned
  2. Authorization: Getting approval for the return (if required)
  3. Return Creation: Creating the purchase return document
  4. Review: Reviewing return details before finalizing
  5. Finalization: Finalizing the return and generating debit note
  6. Stock Adjustment: Automatic stock adjustment
  7. Communication: Sending debit note to supplier
  8. Reconciliation: Reconciling with supplier and payments

Stage 1: Identification

The workflow begins when you identify items that need to be returned:

  • Receiving Inspection: Identify issues during goods receipt
  • Quality Check: Discover quality issues during inspection
  • Inventory Review: Identify excess or unwanted stock
  • Customer Returns: Items returned by customers that need supplier return
  • Documentation: Document the issue with photos, notes, or reports

Stage 2: Authorization

Before creating a return, you may need authorization:

  • Internal Approval: Get approval from management if required
  • Supplier Communication: Inform supplier about the return
  • Return Authorization: Obtain return authorization from supplier if needed
  • Policy Compliance: Ensure return complies with company policies
  • Documentation: Keep records of authorizations and communications

Stage 3: Return Creation

Create the purchase return document:

  1. Navigate to Purchase module â†’ Purchase Return
  2. Click "Create New" or "Add Return"
  3. Select the original purchase bill
  4. Choose items to return and enter quantities
  5. Select return type and reason
  6. Add notes and attach documents
  7. Review calculations (subtotal, tax, grand total)
  8. Save as Draft or Final

Stage 4: Review

Before finalizing, review the return:

  • Items Check: Verify correct items and quantities
  • Amount Verification: Check that amounts are correct
  • Tax Review: Verify tax calculations
  • Reason Validation: Ensure return reason is appropriate
  • Documentation: Confirm all supporting documents are attached
  • Supplier Details: Verify supplier information is correct

Stage 5: Finalization

Finalize the purchase return:

  • Status Change: Change status from Draft to Final
  • Debit Note Generation: System automatically generates debit note
  • Account Update: Supplier account is debited
  • Stock Adjustment: Stock is automatically adjusted
  • Record Creation: Return record is finalized in the system

Stage 6: Stock Adjustment

Automatic stock adjustment occurs:

  • Immediate Update: Stock levels are updated immediately
  • Quantity Deduction: Returned quantities are deducted from inventory
  • Purchase Details: Purchase details quantities are updated
  • History Recording: Stock movement is recorded in history
  • Verification: Verify stock adjustments are correct

Stage 7: Communication

Communicate the return to the supplier:

  • Debit Note: Print or email the debit note to supplier
  • Supporting Documents: Send any supporting documentation
  • Return Instructions: Provide return shipping instructions if needed
  • Follow-up: Follow up with supplier on return status
  • Confirmation: Confirm supplier received the debit note

Stage 8: Reconciliation

Reconcile the return with supplier and payments:

  • Supplier Statement: Verify return appears on supplier statement
  • Payment Adjustment: Use debit note to adjust supplier payments
  • Outstanding Balance: Verify supplier outstanding balance is updated
  • Tax Reconciliation: Reconcile tax adjustments if applicable
  • Documentation: Keep records of reconciliation

Draft Workflow

Using Draft status for preparation:

  • Preparation: Create return in Draft status to prepare without affecting stock
  • Review: Review and modify draft returns as needed
  • Approval: Get approvals while in Draft status
  • Finalization: Finalize when ready to process the return
  • Flexibility: Draft status provides flexibility in return processing

Partial Return Workflow

Workflow for partial returns:

  • Item Selection: Select only items that need to be returned
  • Quantity Entry: Enter partial quantities if returning less than full amount
  • Multiple Returns: Create additional returns for remaining items if needed
  • Tracking: System tracks total returned amount per purchase bill
  • Reconciliation: Reconcile partial returns with original purchase

Direct Return Workflow

Workflow for direct returns (without purchase bill reference):

  • Direct Return Option: Select "Direct Return" when creating return
  • Manual Entry: Manually enter supplier and item details
  • Documentation: Ensure proper documentation for direct returns
  • Stock Adjustment: Stock is still adjusted if stock tracking is enabled
  • Debit Note: Debit note is generated without purchase bill reference

Cancellation Workflow

If you need to cancel a purchase return:

  • Cancellation: Cancel the return from the return details page
  • Stock Restoration: Stock that was deducted is automatically restored
  • Account Reversal: Supplier account debit is reversed
  • Debit Note: Debit note is effectively voided
  • Documentation: Cancellation is recorded in system history

Workflow Best Practices

  • Timely Processing: Process returns promptly after identification
  • Documentation: Maintain complete documentation throughout the workflow
  • Communication: Keep suppliers informed about returns
  • Verification: Verify each stage before proceeding to the next
  • Review: Regularly review return workflow for improvements
  • Training: Train staff on proper return workflow procedures
  • Standardization: Standardize return processes for consistency

Workflow Automation

EquiBillBook automates several workflow steps:

  • Debit Note Generation: Automatically generated upon finalization
  • Stock Adjustment: Automatically adjusted when return is finalized
  • Account Updates: Supplier accounts automatically updated
  • Calculations: All calculations performed automatically
  • History Tracking: All actions recorded in system history

Workflow Monitoring

Monitor the return workflow:

  • Return Status: Track return status (Draft, Final, Cancelled)
  • Reports: Use purchase return reports to monitor workflow
  • Dashboard: View return statistics on dashboard
  • Alerts: Set up alerts for pending returns if available
  • Analytics: Analyze return workflow patterns

Integration with Other Workflows

Purchase return workflow integrates with:

  • Purchase Workflow: Linked to original purchase bills
  • Payment Workflow: Affects supplier payment calculations
  • Inventory Workflow: Triggers stock adjustments
  • Accounting Workflow: Creates accounting entries
  • Reporting Workflow: Appears in various reports

Troubleshooting Workflow Issues

Return stuck in Draft status?

  • Review return details for completeness
  • Verify all required fields are filled
  • Check for validation errors
  • Ensure you have permissions to finalize returns

Workflow not completing?

  • Check each stage of the workflow
  • Verify system settings are configured correctly
  • Review error messages or system logs
  • Contact support if workflow issues persist

Need to modify workflow?

  • Cancel and recreate return if not finalized
  • Review company policies on return modifications
  • Document any workflow changes
  • Train staff on modified procedures

Workflow Optimization

Optimize your return workflow:

  • Streamline Processes: Identify and eliminate unnecessary steps
  • Automation: Leverage system automation features
  • Training: Ensure staff are well-trained on workflow
  • Documentation: Maintain clear workflow documentation
  • Feedback: Gather feedback from staff on workflow improvements
  • Regular Review: Regularly review and improve workflow

Understanding and following the purchase return workflow ensures efficient processing, accurate record-keeping, and proper reconciliation. A well-managed return workflow contributes to better supplier relationships and accurate inventory and financial management.
